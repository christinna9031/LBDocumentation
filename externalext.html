External Apps Extensions

Twitter

Creator: Christinna
Download: Discord Link
Description: This extension lets you post and read Tweets.

Creating Twitter Developer Account

First you will need to create a Twitter developer account and request API access. This process is usually fast, I tested it on several Twitter accounts and got always approved within 24h.

  1. Go to https://developer.twitter.com/en/apply/user.html to apply.
  2. Primary reason should be Hobbyist - building tools for Twitter users. Press Next.
  3. Confirm and fill out all the necessary information. Press Next.
  4. You will need to fill out bunch of stuff on this screen:
    • How will you use Twitter API? - describe in your own words that you're using it for a Twitch plugin to post your stream updates and/or to read recent Tweets of your followers
    • Are you planning to analyze Twitter data - no
    • Will your app use Tweet, Retweet, like, follow or Direct Message functionality? - yes
    • Describe in your own words that you will be using it to post your Twitch stream updates.
    • Do you plan to display Tweets or aggregate data about Twitter content outside of Twitter? - yes or no (depends on whether you are going to use the Recent Tweet feature as well)
    • Will your product, service or analysis make Twitter content or derived information available to a government entity? - no
    • Press Next.

  5. Is everything correct? Press Looks Good.

  6. Review and accept Developer Agreement and Submit Application.
  7. You should get a purple screen You did it! asking you to confirm your email address.
  8. After confirming your email address, you should get another purple screen saying Application Under Review. The approval process usually takes less than 24h.
  9. Once you get approved, go to https://developer.twitter.com/en/portal/projects-and-apps. Click on + Create App.
  10. Name it whatever you like and press Complete.
  11. Copy your API key and API secret key and save it somewhere. Click on the blue button App Settings.
  12. You should see a page with your App Details and App permissions settings (you can also navigate there by going to Projects & Apps - Overview - Your App name).
    1. Click on the blue button Edit in App Permissions Settings and change it to Read and Write. Press Save.
    2. Click on Keys and tokens (right under the page title):
      1. Click on Generate your Access token & secret.
      2. You will get a pop up window containing your access token and secret. Copy them both and save them somewhere.
      3. Click on Yes, I saved them. If you manage to lose them, you can always click on Regenerate.

Initializing the extension

At this point you are ready to initialize the extension. Install it and go to the premade deck.

  1. Click on the Tweet Init button - Edit Commands.
  2. Fill out all the fields in the Send to Extension command. Hopefully you saved them all earlier, so you can just copy paste them one by one. Press done. The button will now trigger every time you restart Transmitter.

How to Tweet

Right click on the Tweet button - Edit Commands.

  • type = Post a Tweet
  • text_username = your Tweet content. Can contain links as well.
  • you can leave everything else empty.

Press done Twice and test your Tweet button. If you set up everything right, you should get a small yellow message saying "Tweet has been posted!".

If you get a message "Error posting Tweet" or "Something went wrong posting the Tweet.", make sure you copied all your tokens and secrets right and followed the instructions (especially the one where you allow the app to read and write).

IMPORTANT
Make sure to not spam it. Twitter is pretty strict and actually blocked my account while I was testing the extension and accidentally sent out 10+ Tweets with the same content. If this happens, you will get a small yellow message saying "Too many similar Tweets. Change the text and/or format and try again later.".

How to retrieve recent Tweet(s)

Right click on the recent tweet button - Edit Commands.

  • type = recent tweet
  • variable = variable name or stack name you want the tweet(s) to be saved under
  • incl_retweets = whether you want to include retweets
  • incl_replies = whether you want to include replies
  • count = how many recent tweets you want to retrieve
  • text_username = Twitter name of the person you want to retrieve tweets from

If count < 1, the most recent tweet will be saved in a variable. If count > 1, the most recent tweets will be saved in a stack.
If you select false in the incl_retweets or incl_replies field, remember that they will still count towards the total amount of tweets retrieved (if count=10, incl_retweets = false and the Twitter account has 2 recent tweets + 8 recent retweets, you will receive only 2 tweets back)

Some tweets contain special characters that LB doesn't like. I created a filter to eliminate them, but it's quite difficult to test them all, so it might still ocassionally give you an error "Corrupted pubsub packet". If that happens, send the Tweet which broke it my way.
