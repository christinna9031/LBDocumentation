FAQ

How do I do X?

Making your own buttons in LioranBoard is a little bit like programming. It definitely takes some time to learn how everything works. Don't expect to be able to create cofmplex buttons right away, but rather learn what each command does and try making something as simple as switching scenes. You can then slowly transition to more advanced buttons.
We welcome new members on our Discord server to ask questions if they encounter a problem with their buttons or commands, but discourage asking for a whole button to be created just for their needs. If you're not interested in learning LB whatsoever and only want the finished product, you can request a commission instead.

Can I modify/resize X?

What you can modify:

  • StreamDeck - window size
  • Deck (in Receiver) - background color (no transparent values), grid size
  • Buttons - background color (no transparent values), border thickness, size

What you cannot modify:

  • LB Receiver - window size
  • Buttons - shape and rounded corners

How to copy a whole deck

Each configs.ini file represents a single deck.

  1. In the main Receiver menu, note the deck number you would like to copy.
  2. Look at the bottom of your Receiver, there is a small grey text that says either 'Local' or 'Appdata'
    • Local - go to Lioranboard Receiver folder and copy the configs.ini file with the same number you noted in step 1.
    • Appdata - go to C:/user/%username/appdata/local/lioranboard and copy the configs.ini file with the same number you noted in step 1.
  3. To make sure you're copying the right deck file, you can open the file and scroll all the way to the bottom to the [configs] section. The name value should be the name of your deck.

How to install or uninstall extensions

Install an extension:

  1. Download the .lbe extension file
  2. Click on Install Extension in your LioranBoard Receiver
  3. Select the extension file you downloaded
  4. Select your default Transmitter you are using. Make 100% sure it is the correct one.
  5. Your Transmitter will automatically refresh, adding a new tab for the extension you just installed. You should also see it added in at the bottom list of all extensions (with checkboxes).
  6. Most extensions include a premade deck with buttons. If you do not see one, create a new button, add "Send to Extensions" command and select the extension you just installed.

Extension is installed properly if:

  • You can see all its fields in your Send to extension command. If you can only see the extension name, the extension was not installed properly or your Transmitter is not running. The amount of input fields depends on the extension, but it should always contain at least one.
  • Your transmitter displays that it's correctly installed (you should see a new tab added).

Uninstall an extension:

  1. Open the original extension .lbe file you wish to uninstall and copy the name in the [extension_name] section. This needs to be an exact match.
  2. Download this file: https://github.com/christinna9031/LioranBoard-Files/releases/download/1.0/uninstall_extension.lbe
  3. Open the file you just downloaded and replace the YOUR EXTENSION NAME YOU WISH TO UNINSTALL with your extension name from step 1.
  4. Follow the same steps for installing an extension.

Math Operators

One thing that is often overlooked is the difference between = (which works only with real values) and =string (which works only with strings). Pay special attention to these two operators, you cannot use them interchangeably as it will break your commands or crash LioranBoard.

Basic operators

Operator Explanation Example
= Equal. x = y x = 7
+= Addition. Same as x = x + y 7 += 2 // 9
-= Subtraction. Same as x = x - y 7 -= 2 // 5
Multiply Multiplication *=. Same as x = x * y 7 *= 2 // 14
Divide Division /=. Same as x = x / y 7 /= 2 // 3.5
DIV Integer division. Same as division, but the remainder is discarded. 7 DIV 2 // 3
MOD Modulus. Returns the remainder of division. 7 MOD 2 // 1
&= Bitwise AND. Same as x = x & y 7 = 000111
2 = 000010
7 &= 2 // 2
^= Bitwise XOR. Same as x = x ^ y 7 = 000111
2 = 000010
7 ^= 2 // 5
| = Bitwise OR. Same as x = x | y 7 = 000111
2 = 000010
7 |= 2 // 7
= String String equal. x = y x =string “hi”
+= String String addition. Same as x = x + y hi +=string there // ”hi there"
Cos Returns a numeric value between -1 and 1, which represents the cosine of the angle given in radians. Cos 90 // -0.45
Sin Returns a numeric value between -1 and 1, which represents the sine of the angle given in radians Sin 90 // 0.89
Tan Returns a numeric value that represents the tangent of the angle. Tan = Sin/Cos. Tan 90 = 1/0 // -2.00

Real vs. string

Real value is a number, while string is text. While a string can contain numbers, a real value cannot contain letters.
If we use Turn to Real command, the following will happen:

  • Turning string “Hello there” to real will result in 0.
  • Turning string “1234” to real will result in 1234.
  • Turning string “Hello there 1234” will result in 0.
  • Turning string “5 Hello there 1234” to real will result in 5.

You cannot execute real operations on strings and string operations on real values.

How to see variables and their values

Press tab in the main LioranBoard Receiver menu. You can then type in the variable name. You can see the stack values if you type the whole name of the stack.

What if I want to use a variable in a string or as a parameter?

Wrap the variable in /$$/. For example, variable name will become /$name$/. Remember that this is done only if the field name is NOT "variable". In this case you want to use the name of the variable without any wrappers.
For example, field "value" can contain either a real value (number) or a string (text). By wrapping the variable in /$$/, you're telling it to work with the value of the variable rather than the variable itself.

Some input fields require you to right click to be able to type inside of them, as left clicking brings up the dropdown menu.



Premade variables

There are a few premade variables in your LB Receiver:

  • current_scene - always shows your current OBS scene
  • previous_scene - always shows your previous OBS scene
  • elapsed_time - time elapsed since LB was launched
  • channel_id(hidden) - Twitch Channel ID. Does not show in the list of variables, but can be passed as a parameter in extension fields.
  • oauth_token(hidden) - Twitch Channel ID. Does not show in the list of variables, but can be passed as a parameter in extension fields.

Button Triggers

LioranBoard has a way of triggering your buttons with different kinds of events.
For example, a button can be set up to trigger every time you get a new subscriber. Or, another button can be set up to trigger after a viewer sends a specific message in your Twitch chat.
Setting up a trigger is as easy as right clicking on a button and selecting Add Twitch Triggers. Do not get confused by the name, as you can set up triggers for non Twitch events (extensions) as well.
We recommend checking Add to Request Queue, which acts as a button queue. If there is another trigger event while the button is still active (for example if you get two subscribers at the same time), it will place it in the request queue and reactivate the button as soon as it's ready again.
You can test your Twitch triggers by clicking on the test buttons in your Transmitter.

Types of triggers:

  • Chat / Whisper messages - listens to all chat messages and triggers when specific conditions are met. Will also listen to whisper messages if you use Twitch: Open Whispers command.
    For example, if your message field is !info, it will trigger every time a viewer sends !info command in chat. Can also contain Wild Cards.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the basic information (viewer's name, full message, emotes and badge list) and/or use String: Wilcard Pull to retrieve your wildcard values.


  • Followers - this requires a standalone extension, as Twitch Pubsub does not send follower alerts. You need to either initiate Follower Alert extension (which is already pre-installed for you) or download Streamlabs Alerts extension, which contains Follower Alerts. The latter sends instantaneous alerts, however you need to have a Streamlabs account connected to your Twitch for it to work.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve the follower's name.
  • Bits - triggers every time a viewer cheers with bits on your stream.
    You can set up a minimum and maximum amount of bits allowed for the trigger to get activated. This way you can have different animations/sounds played depending on how many bits the viewer sent.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (bits amount, viewer's name, viewer's message)

  • Subs - triggers every time you get a new subscriber.
    T1/T2/T3 - regular subscribers (tier1 to tier3)
    Prime - Twitch Prime subscriber
    SubGift - A gifted subscriber
    AnonSubGift - Anonymously gifted subscriber
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (sub name, sub type, recipient for gifted subs, sub message, total number of months subbed etc. )

  • Gift Sub Amount - triggers every time a viewer gifts a subscriber to another viewer.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve name of the gifter, amount of subs gifted and types of tiers.
    If you want to retrieve all the names of the gifted subs, you need to also use Subs trigger (check SubGift and AnonSubGift fields).

  • Host/Raid - triggers every time your stream gets hosted/raided.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (viewer's name, amount of viewers)
  • Point Redeem - triggers every time a viewer redeems a channel point reward. Only works for custom rewards (text or no text)
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (viewer's name, reward's name, message, reward ID)
    If you put * in the Redeem Name field, it will get triggered with any reward. This * does not act as a wildcard (do not use String: Wilcard Pull).
  • Point Redeem with message - triggers every time a viewer redeems a channel point reward. Only works for custom rewards which require viewers to enter text.
    The message field can contain wildcards.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (viewer's name, reward's name, message, reward ID) and/or use String: Wilcard Pull to retrieve your wildcard values.
    If you put a single * in the message field, it will get triggered with any message. This * does not act as a wildcard (do not use String: Wilcard Pull).
  • Extension trigger - controlled by LioranBoard extensions. Can contain wildcards depending on the extension.
    Once the trigger activates the button, you can use Math: Trigger Pull to retrieve all the information (value1, value2, value3) and/or use String: Wilcard Pull to retrieve your wildcard values.
    If an extension uses a trigger, it will be documented in the extension manual.
  • LioranBoard premade triggers - these are all extension triggers covering different LioranBoard and Twitch chat events.
    • !initlioranboard - Every time Lioranboard starts up
    • !reinitializing - When decks are reloaded.
    • !resetlioranboard - When you use the reset button
    • !obsconnected - Lioranboard's first connection to OBS
    • !twitchconnected - Twitch chat first connection
    • !transmitterconnected - Transmitter's first connection
    • !obsalways - Every time LioranBoard connects to OBS
    • !twitchalways - Every time Lioranboard connects to Twitch
    • !transmitteralways - Every time Transmitter connects to Lioranboard
    • !leavescene Scene Name - Every time you switch scene, Math: Trigger Pull 1= transition type, 2= transition duration
    • !enterscene Scene Name - Every time you switch scene, Math: Trigger Pull 1= transition type, 2= transition duration
    • !ban - When someone gets banned, Math: Trigger Pull 1=name of the mod, 2=name of the person who got banned
    • !unban - When someone gets unbanned, Math: Trigger Pull 1=name of the mod, 2=name of the person who got unbanned
    • !timeout - When someone gets timed out, Math: Trigger Pull 1=name of the mod, 2=name of the person who got timed out, 3=duration of the timeout in seconds
    • !untimedout - When someone gets unbanned, Math: Trigger Pull 1=name of the mod, 2=name of the person who got untimed out
    • !clearchat - When a mod clears the chat, Math: Trigger Pull 1=name of the mod.

How command delays work

Command delays are absolute, they’re NOT related to each other. You can think of them as a timeline, where they don't affect each other in any way.
Example 1: If you set up the first command to trigger at 1000ms and the second command to trigger at 1000ms, both commands will trigger at 1000ms, one after another (in this case order matters), exactly 1000ms from when the button was pressed.
Example 2: If you set up your first command to trigger at 5000ms and your second command to trigger at 2000ms, the second command will be triggered exactly 2000ms from when you pressed the button and then first command will be triggered exactly 3000ms after it (5000ms from when you pressed the button).
The command delay is superior to the command order, which means that a second command with a shorter delay will be always triggered before the first command with a longer delay.


How to set up button cooldowns

If you want to add a cooldown to any button, you can use the Overtime command, which artifically increases the duration of the button and prevents it from triggering again.
You can also use Math: Variable Transition + Compare: Skip if command. This will allow you to notify the viewers that the button is on cooldown.


How to trigger buttons with keyboard shortcuts

You can set up hotkeys to trigger LioranBoard buttons by selecting Trigger Shortcut button in the main Receiver menu. Click on the little +, choose the type (only keyboard, gamepad and midi keyboard supported), key + optional modifier and fill in your button ID.

Wild Cards

A wild card * refers to a character that can be substituted for zero or more characters in a string. They are used to search for partial matches instead of exact ones.
Wild cards can be used for Twitch Chat triggers and Redeem points with message triggers. They need to match exactly. You can pull their value by using the String: Wildcard Pull command.



You can use as many wild cards as you want as long as the formatting is matched. Wild card trigger examples:

Trigger Chat Input (green=accepted, red=rejected)
world * world hello world hello hi there hello world hello world hello
* world world hello world hello hi there hello world hello world hello
* world * world hello world hello hi there hello world hello world hello
world * * world hello world hello hi there hello world hello world hello

If you want to allow a single word for your trigger, no matter where it is placed in the sentence, you will need 4 different triggers: word,word *, * word and * word *.
For example, with a trigger !set * to * and a chat message !set timer to 5 , you can pull wild card 1 value (timer) and wild card 2 value (5).

Now, what if you have a command !lights * where the wildcard is a color to set your Philips Hue lights to, but your viewers sometimes include extra text after they specify the color? For example, they send a chat message: !lights red Hello there! and you're only interested in pulling the color name instead of the whole red Hello there! value. In this case, you need to use these two chat triggers: lights * * , lights * (order and spaces matter). Now it will always only pull the color, no matter if there is any additional text.


You can't have 2 wild cards back to back without a space, as it will be turned into just one wildcard.
Twitch automatically removes double space, so you don't have to worry about that.

Controlling OBS from another PC with LB

LioranBoard can connect to your OBS from another PC, as long as they are both sharing the same network.
Follow these steps:

  • Streaming PC:
    1. Run your OBS as normal.
    2. Open CMD and type ipconfig. Write down your IPv4 address. It should start with 192.168.
  • Gaming PC:
    1. Go to your Lioranboard/Receiver folder and open connect.ini. Under [connection] section, add a new entry: websocket_ip="YOUR STREAMING PC IPv4".
    2. Save the file, open Lioranboard and try connecting to the OBS websocket on your streaming PC. It should work as long as nothing (antivirus, firewall) is blocking connection to your streaming PC.

LioranBoard and OBS Streamlabs

LioranBoard natively supports only OBS Studio, as it uses OBS Websocket to communicate. It is possible to make an extension to communicate with OBS Streamlabs, but as of now that hasn't been done yet.

LioranBoard and Elgato Stream Deck

The only way Elgato and LioranBoard can communicate is through key presses. You can trigger LB buttons from your Elgato, but not the other way around. Elgato plugins are unfortunately quite limited, and there is currently no way to trigger Elgato keys within a plugin or via keyboard hotkeys.

How to connect LioranBoard to any app

Lioranboard can connect to any app or service that has public API or websocket access. It is done via custom made extension in JavaScript. If there isn't one you need listed in the Extensions channel in our Discord server, then you will need to create your own or ask in #Suggestions channel for its possible implementation.

How to make your own extension

Lioranboard communicates with extensions via Transmitter (written in HTML and JS) and its own websocket. Information about how to make your own extension can be found in LioranBoard/extension/readme.txt file. You can also go over a few examples of extensions to understand the process.
